## 全体の考え方

このツールは **LTX-Video 2** だけを使います。役割分担は次のとおりです。

| やりたいこと | 指定方法 |
|-------------|---------|
| 日本の道路・車・標識など **見た目** | `references/` の **参照画像** |
| 右折・衝突・カメラの動きなど **動き** | **プロンプト**（英語推奨） |

参照画像あり → **I2V**（Image to Video）  
参照画像なし → **T2V**（Text to Video）

## 起動方法

### 初回以降、毎回やること

**2つのプロセス**が動いている必要があります。

| プロセス | 役割 | URL |
|---------|------|-----|
| ComfyUI | AI エンジン本体 | http://127.0.0.1:8188 |
| Web UI (`app.py`) | 操作画面 | http://127.0.0.1:7860 |

#### ターミナル 1 — ComfyUI

```powershell
cd wan-video-gen
.\scripts\start_comfyui.ps1
```

`Starting ComfyUI at http://127.0.0.1:8188` と出たら **このウィンドウは閉じない**。

> 既に別のターミナルで ComfyUI が動いている場合は、2重起動しない（ポート競合エラーになる）。

#### ターミナル 2 — Web UI

```powershell
cd wan-video-gen
.\scripts\start_ui.ps1
```

ブラウザが http://127.0.0.1:7860 を開きます。開かない場合は手動でアクセスしてください。

### 接続状態の確認

画面上部の **「接続状態」** に次のように表示されます。

- `ComfyUI: 接続済み` → 生成 OK
- `ComfyUI: 未接続` → **「ComfyUI 起動」** を押すか、`start_comfyui.ps1` を実行
- `Ollama: 接続済み` → プロンプト自動拡張（Gemma）が使える
- `Ollama: 未接続` → 拡張は使えないが、英語プロンプトを直書きすれば生成は可能

### 終了方法

1. Web UI のターミナルで `Ctrl + C`
2. ComfyUI のターミナルで `Ctrl + C`

### 各項目の意味

| UI 項目 | 説明 |
|--------|------|
| **モデル** | LTX FP8（速い） / LTX FP16（高品質・重い） |
| **モデルメモ** | 選んだモデルの注意書き |
| **参照画像リストを更新** | `references/` に写真を追加したあと押す |
| **フォルダから選ぶ** | 登録済み参照画像を選択 |
| **またはアップロード** | その場で1枚だけ使う画像を指定 |
| **参照画像の強度** | 0.3〜1.0。高いほど写真に近い見た目（動きは弱くなりやすい） |
| **プロンプト** | 動き・出来事・カメラの説明（**英語推奨**） |
| **Gemma で英語に拡張** | 日本語の短いメモを長い英語プロンプトに変換（Ollama 要） |
| **拡張プレビュー** | 生成前に拡張結果を確認 |
| **拡張後** | 自動拡張の結果。手で直してから生成してもよい |
| **横幅 / 高さ** | 解像度。上げすぎると VRAM 不足 |
| **フレーム数** | 動画の長さ。49 = 約2秒 @ 24fps |
| **シード** | `-1` でランダム。同じ値 + 同条件 = 再現性 |
| **出力フォルダ** | 動画の保存先 |
| **生成動画** | 完成した mp4 のプレビュー |
| **情報** | モード（I2V/T2V）、シード、ファイルパスなど |

## 動画の作り方（基本手順）

### パターン A: 日本の交差点シーン（おすすめ）

交差点の見た目を写真で固定し、プロンプトで事故や動きを指定する流れです。

1. **参照画像を用意**  
   `references/street/` に交差点や車内の jpg/png を入れる

2. **UI を起動**し、接続状態が `ComfyUI: 接続済み` であることを確認

3. **「参照画像リストを更新」** → **フォルダから選ぶ** で写真を選択

4. **参照画像の強度** を `0.65`〜`0.80` あたりに（初回は `0.75`）

5. **プロンプト** に動きだけ書く（日本語でも可）  
   例: `交差点で右折中、左から自転車が飛び出して衝突。車はすぐ止まる。教育的。`

6. **「Gemma で英語に拡張」** にチェック → 必要なら **拡張プレビュー**

7. モデルは **LTX FP8** のまま（まず試すならこちら）

8. **「動画を生成」** を押す

9. 右側に動画が出たら成功。気に入った **シード** をメモしておく

10. 動きが良ければ FP16 に切り替えて本番生成（VRAM に余裕がある場合）

### パターン B: 参照画像なし（プロンプトのみ）

1. 参照画像は選ばない
2. プロンプトに **見た目 + 動き** を英語で長めに書く
3. 生成

日本らしさは弱くなります。**日本シーンなら参照画像ありを強く推奨**します。

### 生成中の表示

- 進捗バーが動く（「ステップ x/y」「残り約 N 秒」）
- FP8 で **30秒〜2分**、FP16 ではそれ以上かかることもある
- 10% 付近でしばらく止まって見えても、多くの場合は正常

### 生成後に確認すること

「情報」欄の **モード**:

- `I2V（参照画像あり）` → 参照画像が効いている
- `T2V（プロンプトのみ）` → 画像は使っていない

## 参照画像の使い方

### フォルダ構成（推奨）

```
wan-video-gen/references/
  street/           … 交差点・道路・横断歩道
  car_interior/     … 運転席・ダッシュボード（車内 POV）
  signs/            … 青標識・道路標識
```

サブフォルダ名は自由です。jpg / png / webp が使えます。

### 写真の選び方

| 良い写真 | 避けたい写真 |
|---------|-------------|
| 明るい・ピントが合っている | 極端に暗い・ブレている |
| 横長（16:9 に近い） | 極端に縦長 |
| 日本の道路・標識がはっきり写っている | 海外の街並み |
| プロンプトと矛盾しない構図 | 車内 POV なのに外観だけ、など |

### 強度の目安

| 強度 | 向いている用途 |
|------|---------------|
| **0.5〜0.6** | 写真はヒント程度。動きを優先したい |
| **0.65〜0.75** | バランス型（**まずここ**） |
| **0.8〜1.0** | 見た目を強く固定。動きが硬くなりやすい |

### 参照画像とプロンプトの役割分担

**良い例**（参照 = 交差点の写真、プロンプト = 動きのみ）:

```
The car begins a right turn.
A bicycle approaches rapidly from the left blind spot and collides with the front-left of the car.
The rider falls. The car brakes immediately. Smooth motion, realistic physics, educational tone.
```

**悪い例**（参照画像があるのにプロンプトでも道路を詳しく書く）:

```
Japanese intersection with zebra crossing, blue road signs, utility poles, left-side traffic,
the car turns right and a bicycle hits...
```

→ 画像とプロンプトが **喧嘩** してブレやすい。

## プロンプトの書き方

### 基本ルール（LTX）

1. **英語**で書く（日本語は Gemma 拡張に任せる）
2. **長め**（50語以上推奨）
3. 参照画像あり → **動き・タイミング・カメラ**中心
4. 参照画像なし → 環境 + 動きの両方

### 構成テンプレート（参照画像あり）

```
[カメラ] Driver POV from inside a Japanese car.
[動き1] The car slows and begins turning right at the intersection.
[動き2] A bicycle suddenly enters from the left blind spot.
[動き3] The bicycle collides with the front-left of the car; the rider falls.
[終わり] The car stops immediately. Smooth motion, realistic physics, educational, no gore.
```

### 日本語から英語へ（Gemma）

1. プロンプト欄に日本語で短く書く
2. 「拡張プレビュー」で確認
3. 変な表現があれば **拡張後** 欄で手直し
4. 生成

Ollama のインストール: https://ollama.com/  
モデル例: `ollama pull gemma:7b` のあと `ollama serve`

### 効果的なキーワード

| カテゴリ | 例 |
|---------|-----|
| 動き | smooth motion, realistic physics, natural movement |
| カメラ | driver POV, dashcam view, steady camera |
| 雰囲気 | educational tone, daytime, cinematic |
| 品質 | high quality, detailed, 4k |

## モデルとパラメータ

### モデル比較

| | LTX FP8 | LTX FP16 |
|---|---------|----------|
| チェックポイント | `ltx-2-19b-distilled-fp8.safetensors` | `ltx-2-19b-distilled.safetensors` |
| 速度 | 速い | 遅い |
| VRAM | 比較的少なくて済む | 多めが必要 |
| 画質 | 良い | より良い |
| デフォルト解像度 | 768×432 | 768×480 |
| デフォルトフレーム | 49 | 65 |

モデルを切り替えると、横幅・高さ・フレーム数・強度の **初期値** も変わります。

### パラメータ一覧

| 項目 | 意味 | FP8 目安 | FP16 目安 |
|------|------|---------|----------|
| 横幅 × 高さ | 解像度 | 768×432 | 768×480 |
| フレーム数 | 長さ（8n+1 推奨） | 49（約2秒） | 65（約2.7秒） |
| 参照画像の強度 | 見た目の固定度 | 0.75 | 0.75 |
| シード | 再現用 | -1（ランダム） | 同左 |

> LTX Distilled は **ステップ数・CFG は UI に出ません**（ワークフロー内で固定）。

### VRAM を節約したいとき

- モデル: **FP8**
- 解像度: **640×384** 程度まで下げる
- フレーム数: **33** や **41**

## おすすめの作業フロー

```
① references/ に日本の写真を追加（随時）
      ↓
② FP8 + 参照画像 + 短いプロンプトで試作（49フレーム）
      ↓
③ 動き・強度・シードを調整して気に入った条件を固定
      ↓
④ 必要なら FP16 + 長めフレームで本番（VRAM に余裕がある場合）
```

**1本の動画を仕上げるコツ**

1. まず **参照画像だけ** 変えて見た目を固める
2. 次に **プロンプト** で動きを調整
3. 良い **シード** が出たらメモして再利用
4. 最後に解像度や FP16 で品質を上げる

## 気を付ける点

### 必ず守りたいこと

- **ComfyUI を2重起動しない**（`Port 8188 is already in use`）
- **LTX は英語プロンプト**（日本語のままだと品質が落ちる）
- **見た目は画像、動きはプロンプト** — 役割を混ぜない
- 生成中は **ComfyUI のターミナルを閉じない**
- モデルファイルは **ダウンロード完了まで中断しない**（破損の原因）

### VRAM が少ない GPU 向け

- 日常は **LTX FP8** を使う
- **FP16 は落ちたら FP8 に戻す**
- 解像度を上げすぎない
- 参照画像あり（I2V）は T2V より VRAM を多く使う

### Git について

- **Git に上げる**: `wan-video-gen` のコード・ワークフロー
- **Git に上げない**: `../ComfyUI/` フォルダ全体、巨大な `.safetensors`、参照写真（任意）

### うまくいかないときのチェックリスト

- [ ] `python scripts/check_models.py` が OK か
- [ ] 接続状態は `ComfyUI: 接続済み` か
- [ ] 参照画像を選んだとき、情報に `I2V` と出ているか
- [ ] プロンプトは英語か（または Gemma 拡張 ON か）
- [ ] FP16 で落ちていないか → FP8 を試す

## CLI（コマンドライン）

```powershell
cd wan-video-gen
.\.venv\Scripts\Activate.ps1

# 接続確認
python scripts\generate.py check

# FP8 + 参照画像
python scripts\generate.py generate "The car turns right..." --image references\street\sample.jpg

python scripts\generate.py --model ltx_fp16 generate "The car turns right..."

python scripts\generate.py generate "..." --image references\street\sample.jpg --seed 42
```

## トラブルシューティング

### ComfyUI に接続できない

**症状:** `ComfyUI: 未接続`、生成ボタンでエラー

**対処:**
1. `.\scripts\start_comfyui.ps1` を別ターミナルで実行
2. UI の **「ComfyUI 起動」** を押す
3. ブラウザで http://127.0.0.1:8188 が開けるか確認

### Port 8188 is already in use

**原因:** ComfyUI が **既に動いている** のに、もう一度起動した

**対処:** 新しく起動しない。UI で「接続済み」ならそのまま使う。  
どうしてもやり直すときは、既存の ComfyUI プロセスを終了してから再起動。

### 400 Bad Request（/prompt）

**原因例:**
- モデルファイルが無い・破損
- ワークフローとモデルの不一致

**対処:**
```powershell
python scripts\check_models.py
.\scripts\download_models.ps1
```

### ComfyUI が生成中に落ちる / ConnectionError

**原因:** ほぼ **VRAM 不足**（特に FP16）

**対処:**
- **LTX FP8** に切り替え
- 解像度・フレーム数を下げる
- 参照画像の強度を少し下げる
- ComfyUI を再起動してキューを空にする

### Gemma 拡張が失敗する

**原因:** Ollama が動いていない

**対処:**
```powershell
ollama serve
ollama pull gemma:7b
```

拡張なしでも、英語プロンプトを直書きすれば生成できます。

### 日本っぽくない / 海外の道路になる

**対処:**
- **参照画像** を日本の実写に差し替える（いちばん効く）
- 参照画像の強度を **0.7** 前後に上げる
- プロンプトでは環境を書きすぎない（動き中心に）

### 動きが硬い / ほとんど動かない

**対処:**
- 参照画像の強度を **0.55〜0.65** に下げる
- プロンプトに `smooth motion`, `natural movement` を足す
- フレーム数を **49 → 65** に増やす（VRAM に注意）

### 生成が遅い・進捗が止まったように見える

- FP8 でも 1〜3 分かかることがある
- 進捗バーの「ステップ x/y」が増えていれば正常
- FP16 はさらに時間がかかる
