## 起動方法

### 毎回やること

1. PowerShell を開く
2. 以下のコマンドを順に実行:

```powershell
cd C:\Users\<ユーザー名>\Projects\wan-video-gen
.\.venv\Scripts\Activate.ps1
python app.py
```

3. 「ComfyUI を起動中...」と表示され、しばらく待つ
4. ブラウザが自動で開く（http://127.0.0.1:7860）
5. 画面が表示されたら準備完了

### 終了方法

- ブラウザのタブを閉じる
- PowerShell で `Ctrl + C` を押す

## 動画を生成する

### 手順

1. **モデル**を選ぶ（初めては「wan」がおすすめ）
2. **プロンプト**に作りたい動画の説明を書く
3. **「動画を生成」**ボタンを押す
4. 進捗バーが動き始める（ステップ数と残り時間が表示される）
5. 完了すると右側に動画が表示される

### 生成にかかる時間

| モデル | 時間の目安 |
|--------|-----------|
| Wan 2.1 | 2〜5分 |
| LTX Distilled | 30秒〜1分 |

## モデルの切り替え

画面上部の「モデル」ドロップダウンで切り替えます。

### Wan 2.1 1.3B（高品質・日本語対応）

- **日本語のプロンプトが使える**
- 画質が良い
- 生成は遅め（30 ステップ）
- 用途: 品質重視の動画、日本語で指示したい時

### LTX-Video 2 Distilled（高速）

- **英語のみ対応**（日本語を入れると品質が下がる）
- 生成がとても速い（8 ステップ）
- プロンプトは長く詳しく書く必要がある
- 用途: 素早く試したい時、大量生成

## パラメータの調整

| パラメータ | 意味 | おすすめ値 |
|-----------|------|-----------|
| **横幅** | 動画の横のピクセル数 | Wan: 832 / LTX: 768 |
| **高さ** | 動画の縦のピクセル数 | Wan: 480 / LTX: 512 |
| **フレーム数** | 多いほど長い動画 | Wan: 81(約5秒) / LTX: 97(約4秒) |
| **ステップ数** | 多いほど高品質だが遅い | Wan: 30 / LTX: 8 |
| **CFG スケール** | プロンプトへの忠実度 | Wan: 6.0 / LTX: 1.0 |
| **シード** | 同じ値=同じ結果 | -1（ランダム） |

> **注意:** 解像度を上げすぎると VRAM 不足（OOM）でエラーになります。

## 出力フォルダの変更

「生成」タブの下部にある「出力フォルダ」欄でパスを変更できます。

```

```

フォルダが存在しない場合は自動で作成されます。

3. 「更新」ボタンを押す
4. 最近生成された動画の一覧が表示される

## CLI（コマンドライン）での操作

ブラウザを使わずコマンドでも操作できます。

```powershell
# ComfyUI 接続確認
python scripts/generate.py check

# Wan で1本生成
python scripts/generate.py generate "猫がビーチを歩く映像"

# LTX で1本生成
python scripts/generate.py --model ltx generate "A cat walking on a beach"

# シード指定
python scripts/generate.py generate "プロンプト" --seed 42

# 出力名を指定
python scripts/generate.py generate "プロンプト" --output my_video
```

## バッチ生成（一括生成）

### CSV ファイルの準備

`prompts/example.csv` を参考に CSV を作成:

```csv
prompt,negative_prompt,seed
"猫がビーチを歩く","blurry, ugly",-1
"東京の夜景、ネオン、雨","blurry, low quality",42
"山の風景、雲が流れる","ugly, deformed",-1
```

### 実行

```powershell
python scripts/generate.py batch --csv prompts/my_prompts.csv
```

各動画が順番に生成され、`output/` に `batch_001.mp4`, `batch_002.mp4`... と保存されます。

## プロンプトのコツ

### 基本構造

```
[何が] + [何をしている] + [どこで] + [どんな雰囲気/スタイル]
```

### 良い例

```
A golden retriever running through a field of sunflowers,
golden hour lighting, cinematic camera movement, 4k quality
```

```
晴れた日の東京渋谷のスクランブル交差点、大勢の人が横断歩道を渡っている、
タイムラプス風、上からの俯瞰アングル
```

### 悪い例

```
猫
```
→ 短すぎ。何をしているか、どんな雰囲気か書かないと品質が低い。

### 効果的なキーワード

| カテゴリ | キーワード例 |
|---------|-------------|
| 品質 | 4k, cinematic, high quality, detailed |
| カメラ | tracking shot, close-up, wide angle, POV |
| 照明 | golden hour, dramatic lighting, soft light |
| スタイル | realistic, anime style, watercolor |
| 動き | slow motion, timelapse, smooth movement |

### ネガティブプロンプト

「こうなってほしくない」を書きます。デフォルトでほぼ OK:

```
blurry, low quality, distorted, watermark, ugly, deformed
```

## トラブルシューティング

### ComfyUI に接続できない

**原因:** ComfyUI が起動していない

**対処:**
1. `python app.py` で自動起動を待つ（最大2分）
2. それでもダメなら別ターミナルで手動起動:
   ```powershell
   cd ..\ComfyUI
   .\venv\Scripts\python.exe main.py --listen 127.0.0.1 --port 8188
   ```

### VRAM 不足（OOM エラー）

**原因:** 解像度が高すぎる / フレーム数が多すぎる

**対処:**
- 横幅・高さを下げる（Wan: 640×384、LTX: 512×320）
- フレーム数を減らす（33〜49）
- LTX の FP8 版を使う（デフォルト設定）

### 生成された動画の品質が低い

**対処:**
- プロンプトをより詳細に書く
- LTX の場合は英語で長文にする
- ステップ数を増やす（ただし遅くなる）
- CFG 値を調整する（Wan: 5〜8 が良い範囲）

### 動画が真っ黒 / ノイズだけ

**原因:** モデルファイルが破損している可能性

**対処:**
- モデルファイルを削除して再ダウンロード
- ダウンロード中に中断すると破損するので、最後まで完了させる

### ブラウザが開かない

**対処:**
手動でブラウザを開いて以下にアクセス:
```
http://127.0.0.1:7860
```
